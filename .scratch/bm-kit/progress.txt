## Codebase Patterns
- `context/workflow.md` is the single shared context file loaded by every skill — update it when the pipeline changes, not each skill individually
- Issue frontmatter fields: `id`, `title`, `status`, `priority` (YAML, not bare key-value)
- Valid status values: `needs-triage`, `needs-info`, `ready-for-agent`, `ready-for-human`, `wontfix`, `done`
- `docs/agents/issue-tracker.md` is the authoritative spec for `.scratch/` format — all skills reference it
